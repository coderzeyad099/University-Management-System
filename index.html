ملخص المشروع

هذا المشروع هو نظام إدارة جامعة متكامل يعمل داخل شبكة محلية أو على سيرفر داخلي. الهدف: تنظيم جميع شؤون الجامعة بطريقة سهلة ومباشرة، تشمل شؤون الطلاب، الجداول، الحضور، المكتبة، والمالية. النظام صُمم ليكون بسيطًا لأي مستخدم — حتى لمن ليس لديهم خبرة تقنية.

من يستفيد؟
الإداريون، أساتذة الكلية، موظفو المالية والمكتبة، وأيضًا الطلاب.
الأمان والخصوصية
صلاحيات مخصصة لكل دور، تسجيل دخول آمن، نسخ احتياطي دوري، وتشفير كلمات المرور.
قابلية التوسعة
تصميم وحداتي (Modular) يسمح بإضافة وحدات مستقبلية بسهولة، مثل بوابة القبول أو التكامل السحابي.
دعم لغات متعددة
يمكن للنظام العمل بالعربية، الإنجليزية، والكردية لتسهيل الاستخدام لجميع الفئات.

الأقسام الرئيسية

شؤون الطلبة

سجل طلابي، بيانات شخصية، حالة دراسية، وثائق.

المكتبة

إدارة كتب، إعارة، غرامات، وإحصائيات الإعارة.

الجداول والحضور

جدولة محاضرات، نظام حضور وربطه بالهوية أو QR.

المالية

دفعات، إيصالات، تقارير مالية وميزانيات بسيطة.

الأرشيف

حفظ مستندات إلكترونيًا وقوالب تقارير للتحميل/الطباعة.

التقارير والمراقبة

لوحات تحكّم وتقارير قابلة للتصدير (CSV / PDF).

التقنيات الحديثة المستخدمة

  • واجهة المستخدم (UI) — تصميم العناصر المرئية: أزرار، نماذج، قوائم؛ لتسهيل التفاعل.
  • تجربة المستخدم (UX) — تحسين شعور المستخدم أثناء التفاعل: سهولة الوصول للمعلومات وتقليل التعقيد.
  • الواجهة الأمامية (Front-end) — HTML, CSS, JavaScript: مسؤول عن الشكل والتفاعل في المتصفح.
  • الخلفية (Back-end) — PHP أو Node.js: مسؤول عن قواعد البيانات، المنطق، وحماية البيانات.
  • قاعدة البيانات (Database) — MySQL أو MariaDB: تخزين بيانات الطلبة والجداول والسجلات.
  • APIs — واجهات برمجية للتواصل بين الأنظمة.
  • التشفير & أمن المعلومات — HTTPS، تشفير كلمات المرور، وسياسات صلاحيات دقيقة.
  • تصميم وحداتي (Modular) — كل ميزة (المكتبة، المالية...) كتجزئة يمكن تطويرها لاحقًا بسهولة.

الشبكات والبنية المطلوبة

الشروط الأساسية لتشغيل النظام بكفاءة داخل الجامعة:

  • شبكة داخلية (LAN) — جميع الأجهزة والسيرفرات على نفس الشبكة.
  • سيرفر محلي — لاستضافة قاعدة البيانات وBack-end.
  • عنوان ثابت داخلي (Static IP) — لتسهيل الوصول للسيرفر.
  • Switch / Router جيد داخل الحرم الجامعي مع إعداد قواعد أمان.
  • نسخ احتياطي للشبكة — خطة لاسترجاع البيانات عند فشل السيرفر أو الشبكة.
توضيح: السيرفر هو "محطة رئيسية"، وأجهزة الكمبيوتر هي "منازل" تحتاج طريقًا واضحًا للوصول.

الصلاحيات (Roles & Permissions)

نظام صلاحيات واضح يضمن لكل مستخدم رؤية ما يسمح له به:

  • Admin (المشرف العام) — كامل الصلاحيات على جميع الأقسام.
  • Dept Manager (مدير القسم) — إدارة قسمه فقط: تعديل الجداول، منح الدرجات، إعداد التقارير.
  • Staff (الموظف) — إدخال البيانات اليومية ضمن القسم المخصص له.
  • Student (الطالب) — عرض بياناته، جدول المواد، ونتائج الدرجات الخاصة به فقط.

UI / UX — تجربة سهلة للمستخدم

  • بساطة الواجهة — شاشات واضحة وأزرار بارزة.
  • توجيه المستخدم — خطوات منطقية مع رسائل توضيحية.
  • استجابة سريعة — تحميل الصفحات بسرعة، وإظهار مؤشر عند العمليات الطويلة.
  • الوصولية — تكبير خطوط، تباين ألوان مناسب، ودعم قارئات الشاشة.
  • اختبارات المستخدم — تجارب من قبل موظفين وطلاب قبل الإطلاق.

الأمان والخصوصية

  • تشفير كلمات المرور (bcrypt)
  • التحقق من صحة المدخلات لمنع SQL Injection و XSS
  • جلسات آمنة وانتهاء صلاحية تلقائي
  • سجل نشاط (Audit Log) لتتبع التغييرات الإدارية

خطة العمل والمدة التقديرية

التقدير لفريق صغير (1-3 مطورين + مصمم + مسؤول شبكة):

الأسبوع 1
التحليل والمتطلبات — مقابلات مع المستخدمين وتحديد الميزات الأساسية.
الأسبوع 2-3
تصميم UI/UX — Wireframes وصفحات أساسية ومراجعة مع الجامعة.
الأسبوع 4-8
التطوير — Back-end و Front-end، قواعد البيانات، واجهات الإدارة.
الأسبوع 9-10
الاختبار والنسخ التجريبي — اختبارات وظيفية وأمان وتحميل.
الأسبوع 11
النشر والتدريب — نشر رسمي داخل الشبكة وتدريب الموظفين.
الأسبوع 12+
دعم وتحسين — إصلاح الأخطاء وإضافة تحسينات.

الخلاصة والهدف
